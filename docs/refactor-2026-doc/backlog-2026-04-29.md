## P1 — mdBook documentation polish

### P1.1 — Collapse `Techniques` sections by default in the SUMMARY

The current `docs/SUMMARY.md` lists every per-technique page under
each area as a child bullet — mdBook renders all of them expanded
in the sidebar, which buries the lower areas.

Root cause: mdBook's fold check is `section_number.len() > level`
(strict `>`). With the previous `level = 1`, chapters at depth 1
were NOT folded (`1 > 1` false) so the area pages rendered
expanded. Bumping to `level = 0` (strict `>`) folds every
chapter at depth ≥ 1 that has sub-items — the Techniques area
pages — while childless top-level entries (Getting Started, By
Role, Reference) stay visible because mdBook's fold path requires
`has_sub_items` to be true.

No SUMMARY restructure needed — the existing tree already has the
right shape; only the threshold was wrong.
