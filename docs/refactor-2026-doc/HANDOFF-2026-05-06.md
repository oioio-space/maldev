## Don't lose

- `MALDEV_PACKER_RUN_E2E=1` is the production gate for actually
  jumping to OEP. Tests never set it; production builds opt in.
- notepad.exe FAILS to reflective-load (COMCTL32 SxS ordinal
  redirection). Use xcopy/where/find for E2E tests. Activation-
  context support is Phase 1f territory.
- `pe/packer/runtime` is Windows-only today. Linux ELF stub
  returns `ErrNotWindows` — Phase 1f mirror.
- `pe/packer.Options` will gain `Pipeline []PipelineStep` in
  Phase 1c. Backward compat: zero-pipeline = current AES-GCM
  behaviour. Existing tests must continue to pass.
