## Don't lose

- `MALDEV_PACKER_RUN_E2E=1` is the production gate for actually
  jumping to OEP. Tests never set it; production builds opt in.
- notepad.exe FAILS to reflective-load (COMCTL32 SxS ordinal
  redirection). Use xcopy/where/find for E2E tests. Activation-
  context support is Phase 1f territory.
- `pe/packer/runtime` is Windows-only today. Linux ELF stub
  returns `ErrNotWindows` — Phase 1f mirror.
- `pe/packer.Options` will gain `Pipeline []PipelineStep` in
  Phase 1c. Backward compat: zero-pipeline = current AES-GCM
  behaviour. Existing tests must continue to pass.

## Phase 1d shipped — anti-entropy ✅

`pe/packer/entropy.go` (~360 LOC) + `entropy_test.go` (16 tests).
New pipeline op `OpEntropyCover` (= 4) with three algorithms:

| Algo | Apparent entropy | Size | Use case |
|---|---|---|---|
| `EntropyCoverInterleave` | ~7.4 bits/byte (default 33% pad) | +50% | Real Shannon drop via low-entropy pad spliced between ciphertext chunks; pad pattern selectable (zeros / int3 / nop / mixed-asm). |
| `EntropyCoverCarrier` | unchanged bulk; first 32 bytes match PNG | +32B | Defeats "first-bytes look random" droppers heuristics. |
| `EntropyCoverHexAlphabet` | ≤4 bits/byte | ×2 | Each byte → 2 alphabet bytes (16-byte code-like alphabet: REX / mov / int3 / call / nop). Reversible via inverse LUT. |

**Operational stack to land below 5 bits/byte:** Interleave →
HexAlphabet, verified at ~4 bits/byte by `TestEntropyCover_StackedDeepDrop`.

The honest math correction shipped in `packer-design.md`:
the original "~4-5 bits/byte" claim for Interleave alone was
optimistic — Shannon caps a 33% mix at ~7.4. Stacking with
HexAlphabet is the operational answer.
