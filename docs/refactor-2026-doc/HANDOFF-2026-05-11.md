## Walker suite ‚Äî final disposition

The 7-slice walker suite I scoped this morning (~1820 LOC,
v0.105.0‚Äìv0.111.0) is now classified as **on-demand only**.
Each future slice ships only when an operator hits a real
payload that fails for a specific directory's staleness.
Trigger conditions documented in
`docs/refactor-2026-doc/packer-2f3c-walker-suite-plan.md`:
- 2-F-3-c-3 EXCEPTION ‚Äî only when C/C++ payload uses SEH
- 2-F-3-c-4 LOAD_CONFIG ‚Äî only when payload has CFG (and even
  then, CFG fails vanilla too ‚Äî likely no-fix)
- 2-F-3-c-5 EXPORT ‚Äî only when packing DLLs
- 2-F-3-c-6 RESOURCE ‚Äî only when payload calls `FindResource`
- 2-F-3-c-7 DEBUG ‚Äî purely cosmetic for tools
- 2-F-3-c-8 DELAY_IMPORT ‚Äî only for `/DELAYLOAD` payloads
